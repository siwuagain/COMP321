Sample data will be taken from `./sample/'.  All pairs of matching
(.in, .ans) files there will be used as sample data, in alphabetical
order by file name.  This data will be displayed in the problem
statement.

Secret test data will be taken from `./secret/'. All pairs of matching
(.in, .ans) files there will be used to test each
submission. Optionally a text file (with filename extension .txt)
describing the purpose of an input file may be present. For each .txt
file there must be a matching (.in,.ans) pair.

greedy:
This file tests the case where solving the first trap and none else 
is the optimal solution before solving the final trap.

inorder:
This file tests the case where the optimal solution is to solve all the traps in order.

mixed:
This file tests the case where it is optimal to solve up to a certain trap, then skip to the final trap
and then solve the rest.

secret_{1, 2, ... ,20}:
This file's content is randomly generated, to ensure comprehensive coverage
